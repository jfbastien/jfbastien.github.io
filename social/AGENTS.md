# Open Graph Image Instructions

`social/generate.ts` creates `og.png`.

The OG image should read as a business card for the same dossier system, not as
a generic social poster. It may be more graphic than the page, but it should use
the same form-row, dot-leader, paper, and Berkeley Mono vocabulary.

Prefer fields that match the masthead: name, work, location, and URI. Contact
is optional; the preview card does not need every Dispatch field.
Avoid duplicate URLs and implementation labels.
If a portrait is used, treat it as a restrained ID-card field, not as a hero
image.
OG register rows may be rendered as fixed static rows when that better preserves
the counted-dot grid.

If OG text uses glyphs not covered by Berkeley Mono, include the supplement font
in the Satori font list.
